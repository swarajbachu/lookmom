lookmom · product plan

GitHub Orgs, first-class

Connect an org once. Members only prove who they are on GitHub — no 50-person SAML dance. Artifacts live in the org library.

01 · The pain

  • Every viewer needing org OAuth / SAML approval
  • Typing org slugs instead of picking connected orgs
  • No place that lists “what’s in Aperturs”

02 · Two connect levels

Identity

Everyone. Proves @login. Scopes: read:user (+ email). No org grant.

Org link

One linker. Scopes: read:org. Lists members, refreshes roster, powers the org library.

03 · Flows

  1. Linker → Orgs → connect GitHub org → pick org → sync members
  2. Owner → share / publish artifact into that org
  3. Member → connect identity GitHub only → sees orgs they’re on the roster for → opens artifacts
  4. Fallback → hidden email? GitHub username match or live check with linker token

04 · Screens

Nav:  Gallery · Orgs · you@… · Log out

/orgs              → all orgs I can see
/orgs/:slug        → artifacts | members | sync
/a/:id             → viewer (+ share → pick linked org)
/gallery           → mine (org badges)

05 · Access rules

  1. Public / owner / email allowlist
  2. Email or @login on org roster
  3. Live membership via linker token (not each viewer)
  4. Else deny — “ask someone to re-sync the org”

06 · Phases

PhaseShip
1Org links + org roster + gate
2/orgs UI, share to linked orgs
3Identity-only connect + auto org list
4Teams polish, CLI, encrypt tokens

07 · Defaults

  • Member list visible to org viewers
  • One primary linker per org
  • Optional team as hard filter
  • Migrate github_team → org-owned artifacts
