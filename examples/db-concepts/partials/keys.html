Keys link rows across tables

A primary key uniquely identifies a row. A foreign key points at another table’s key — that’s how relationships are expressed without duplicating whole records.

identity
Primary key (PK)

Stable, unique, never null. Often a surrogate like id or a natural key like email when truly immutable.

Foreign key (FK)

orders.user_id → users.id. Enables joins and (optionally) cascade rules when a parent row is deleted or updated.
